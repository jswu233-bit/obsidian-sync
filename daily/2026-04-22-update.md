### 1) Claude Design 正式进入研究预览：Anthropic 不只卖模型，开始直接卖“设计产出”
- **发生了什么**：Anthropic Labs 上线了 `Claude Design` 研究预览版，支持通过对话生成设计稿、交互原型、演示文稿和单页页面，并可在聊天与右侧画布之间反复迭代。官方帮助文档披露，它默认继承组织级 design system，支持注释式修改、版本管理与导出分享；订阅计费也与 Claude Chat / Claude Code 分开，采用单独周配额与额外 usage 机制。36氪《设计行业的“棺材板”，要被Claude Design盖上了》则补充了更市场化的解读：这个入口实质上把“想法 → 视觉稿 → 继续交给 Claude Code 开发”的链路开始串起来。
- **为什么重要**：这不是简单的“给模型加个出图按钮”，而是 Anthropic 正在把模型能力包装成高频业务工作台。对 PM、创业团队和小型产品组织来说，最有价值的不是一张图，而是把“需求表达、风格约束、版本迭代、交接开发”放进一个闭环里。如果这个链路跑顺，竞争对象就不再只是 OpenAI、Google，而会直接碰撞 Figma、Canva、PPT、低端外包设计服务。
- **Zoe点评**：**看多方向，看空短期使用成本。** 方向上我很明确：Anthropic 正在从“最好用的模型”往“最先占住具体工作成果入口”走，这比再刷一次 benchmark 更值钱；但短期我不建议你把它当成熟生产工具大规模押注，原因很简单——配额独立、消耗高、流程还在 preview，今天更适合拿来验证“AI 设计工作台”是否能吃掉你们内部一部分原型/汇报页流程，而不是直接替代完整设计体系。

### 2) WaytoAGI 知识库本周论文精选：Agent 的真正分水岭，正在从“更会回答”转向“更会长期工作”
- **发生了什么**：飞书知识库本周 AI 论文精选集中在一条很清晰的主线：Anthropic 的自动化对齐研究员（AAR）把弱监督到强监督的 performance gap recovered 在五天内从人工七天做到接近补齐；AiScientist 用 `thin control, thick state` 与 File-as-Bus 把长时程科研 agent 的状态管理问题系统化；AlphaEval 则把评测对象从 model API 推向真实 commercial agent product；另有 Nemotron 3 Super、Memory Transfer Learning、Auto-Diagnose、WebXSkill、Muses-Bench 等工作，分别从长上下文、高吞吐、跨任务记忆迁移、真实工程诊断、web skill 复用和多主体协作等角度补足 agent 的生产级短板。
- **为什么重要**：这些论文串起来看，行业共识已经越来越明确：**决定 agent 能不能真正落地的，不再只是“单轮推理多强”，而是状态管理、环境记忆、工作流复用、可验证评测和与真实组织流程的耦合能力。** 换句话说，下一阶段的壁垒不只在 base model，而在“模型 + 持久状态 + 环境 scaffold + 可操作评估”的系统设计。
- **Zoe点评**：**强烈看多这条“系统能力栈”路线，继续上调优先级。** 对 Jamie 来说，今天最该跟的不是某个又提分 1.5 的榜单，而是这些论文背后共同指出的产品方向：谁能把 memory、tooling、evaluation、workflow 变成一体化产品，谁就会吃掉下一波企业级 agent 预算。明天如果要继续跟 OpenClaw / Hermes / Claude Code 这类产品，重点就盯“状态是否持久”“工作是否可复用”“结果是否可验证”。

## 🦞 OpenClaw和其他agent新闻融合板块

### 1) Hermes Agent 的热度不是因为“功能比 OpenClaw 多一倍”，而是把复杂度藏进了默认规则里
- **发生了什么**：36氪《Hermes 凭什么两个月接棒 OpenClaw？》把 Hermes 与 OpenClaw 做了很直白的拆解：表面功能表几乎高度重合——调度、子 Agent、浏览器自动化、TTS、Vision、多平台消息集成，两边都有。真正的差异点不在功能目录，而在默认体验：Hermes 把 Skill 自动生成、技能离线进化、高频记忆写入、全文检索等能力做成了几乎“放着就会自己发生”的系统，而 OpenClaw 更多保留显式配置、手工控制与可理解性。
- **为什么重要**：这篇文章其实点到了 agent 产品竞争里一个特别关键的分水岭：**用户感知到的“聪明”，很多时候并不是模型更强，而是产品把本该由用户判断的复杂度预先做掉了。** Hermes 赢的不是“能力不存在于 OpenClaw”，而是“默认替用户做了更多选择”。这对 agent 产品设计太关键了。
- **Zoe点评**：**我看多“默认替用户完成更多判断”的产品路线，但不看多无边界黑盒化。** 对 Jamie 的语境里，这意味着 OpenClaw 如果要继续拉开差距，关键不是再加 3 个工具，而是把“何时存记忆、何时触发复盘、何时启用技能、何时提醒用户”做成一套更顺手的默认行为。Hermes 证明了：省心本身就是产品力。可我也不建议无脑学 Hermes 的黑盒味道——OpenClaw 的可控、可读、可编排，依然是它在 power user 里的核心资产。

### 2) Agent 竞争的下一步，不是“会不会调工具”，而是“会不会把项目记住并继续干下去”
- **发生了什么**：不管是 Hermes 对技能演化与主动记忆的强调，还是本周 AI 论文里 AiScientist / Memory Transfer Learning / WebXSkill 的结果，都在指向同一件事：agent 正从“聪明助手”走向“可持续工作的系统”。今天真正被重视的能力，不再是一次性把问题答得漂亮，而是跨小时、跨会话、跨任务持续积累状态与经验。
- **为什么重要**：这直接对应到 OpenClaw 的产品机会。OpenClaw 已经有 cron、memory 文件、sub-agent、gateway、多渠道消息和工作区文件系统，这些都是做“长期工作体”而不是“聊天窗口”的基础设施。现在差的更多是默认体验、策略自动化和把这些能力压缩成用户能无痛感知的产品动作。
- **Zoe点评**：**明确建议继续加码“长期工作体”而不是“万能聊天体”。** 如果 Jamie 在想 OpenClaw 后面该往哪推，我的判断很硬：优先级应该放在 memory / cron / session / delegation 的衔接体验上，而不是再追一个纯聊天方向的“更像人”。Agent 市场会越来越奖励“能替你把事持续干完”的产品。
