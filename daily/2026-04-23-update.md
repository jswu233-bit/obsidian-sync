## 🤖 AI新闻板块

### 1) Claude Design 研究预览上线：Anthropic 开始直接吃“设计产出”入口
- **发生了什么**：Anthropic 正式开放 `Claude Design` 研究预览，支持通过自然语言生成设计稿、交互原型、演示文稿和单页页面，并能在聊天与画布之间来回迭代。官方帮助文档还明确了它会继承组织级 design system，支持导出与分享，并且配额与 Claude Chat / Claude Code 分开计算。36 氪《设计行业的“棺材板”，要被 Claude Design 盖上了》则把这件事翻成了更直白的商业语言：Anthropic 已经不满足于卖模型，开始直接占产品设计工作台。
- **为什么重要**：这不是简单的“多了个生图能力”，而是 Anthropic 在切一个高频、高价值的中间协作层：需求表达、原型确认、汇报材料、设计交接。这类工作过去分散在 PM、设计、前端和 PPT 工具之间，现在开始被一个 AI 入口压缩。
- **Zoe点评**：**明确看多方向，但不建议短期神化成“Figma 替代品”。** 真正值钱的是它能不能吃掉“第一版方案”和“中间协作成本”，不是能不能立刻替掉专业设计师。对 Jamie 来说，这条线值得持续上调优先级，因为它直接关系到产品工作流是否会被重写。
- **链接**：

  - 官方计费：https://support.claude.com/en/articles/14667344-claude-design-subscription-usage-and-pricing

### 2) WaytoAGI 本周论文精选：Agent 真正的分水岭在“长期工作能力”
- **发生了什么**：WaytoAGI 飞书知识库本周 AI 论文精选聚焦在几条非常一致的主线：Anthropic 的 AAR（自动化对齐研究员）把弱监督到强监督的 performance gap 在 5 天内追回到 0.97；AiScientist 通过 `thin control, thick state` 和 File-as-Bus 机制，把长时程科研 agent 的状态管理问题系统化；AlphaEval 则把评测对象从模型 API 推到真实商业 agent 产品。除此之外，Nemotron 3 Super、记忆迁移学习、Auto-Diagnose、WebXSkill、Muses-Bench 也都围绕长上下文、持久状态、跨任务复用与真实工作流评测展开。
- **为什么重要**：这些论文放在一起看，行业共识已经非常清楚：下一阶段 agent 的壁垒，不是“单轮回答再强一点”，而是“能不能把状态记住、把工作继续做、把结果验证掉”。
- **Zoe点评**：**强烈看多“系统能力栈”路线，继续上调优先级。** Jamie 后面如果还要盯 OpenClaw、Hermes、Claude Code 这类产品，我建议少看 benchmark，重点盯四个词：`memory / workflow reuse / environment scaffold / evaluation`。这四件事比单次聪明更值钱。

## 🦞 OpenClaw和其他agent新闻融合板块

### 1) Hermes 的热度，本质上是“默认替用户做决定”赢了
- **发生了什么**：36 氪《Hermes 凭什么两个月接棒 OpenClaw？》拆得很清楚：Hermes 和 OpenClaw 在功能表上高度重合，调度、子 Agent、浏览器自动化、Vision、TTS、多平台消息集成几乎都不缺。真正拉开体感差异的，不是有没有这些功能，而是谁把复杂度藏得更深、默认动作做得更多。
- **为什么重要**：Agent 产品竞争已经进入一个非常现实的阶段——用户未必在意你有没有第 21 个工具，但非常在意你是不是不用教就能顺手干活。
- **Zoe点评**：**我看多“默认自动化体验”，但不看多无边界黑盒化。** 对 OpenClaw 这条线，我的判断很明确：该补的是“何时记忆、何时提醒、何时复盘、何时委派”的默认策略，而不是继续堆功能目录。Hermes 证明了省心本身就是产品力，但 OpenClaw 的可控、可读、可编排依然是重要资产，别丢。
- **链接**：https://36kr.com/p/3767963450196480

### 2) Agent 的下一场战争，是“长期工作体”而不是“万能聊天体”
- **发生了什么**：不管是 Hermes 对技能进化、主动记忆和全文检索的强调，还是本周论文对 persistent memory、File-as-Bus、跨任务记忆迁移的验证，都在说明一件事：用户开始把 agent 当作长期工作系统，而不是一次性聊天窗口。
- **为什么重要**：这和 OpenClaw 的天然优势是对得上的。cron、memory 文件、sub-agent、workspace、gateway、多渠道消息，本来就更像“会继续工作的身体”，不是一个单纯会说话的模型壳子。
- **Zoe点评**：**建议继续加码“长期工作体”路线，暂停追逐泛聊天方向的热闹。** 如果 Jamie 要看产品机会，我会把优先级压在 `memory × session × delegation × reminder` 的闭环上。这条线更像真钱机会。

### 3) 生态侧冒出新信号：多 Agent 协作开始从概念走向产品化
- **发生了什么**：@op7418 今天公开页里提到一个叫 **Bloome** 的产品，可以把本地 Agent 和线上 Agent 拉进一个群里协作，示例里甚至直接点名了 OpenClaw、Claude Code、Codex 这类能力体一起协作。这个方向虽然还早，但很像在把“Agent 之间怎么协作”产品化。

- **Zoe点评**：**看多这个方向，但短期更适合观察，不建议立即重押单一产品。** 价值点不在“拉群”这个形态，而在它是不是能稳定处理跨 Agent 上下文、权限和角色分工。如果能，后面会很有戏。
