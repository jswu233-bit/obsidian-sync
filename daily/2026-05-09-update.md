### 5. When to Retrieve During Reasoning
**发生了什么**：传统 RAG 通常在推理前只检索一次，但长链推理模型经常在中途才暴露知识缺口。ReaLM-Retrieve 让系统在推理过程中动态判断**什么时候该检索**，并按步骤注入证据。在 MuSiQue、HotpotQA 和 2WikiMultiHopQA 上，相比标准 RAG 带来 **+10.1% 绝对 F1**，同时比固定间隔检索减少 **47% 检索调用**。

**为什么重要**：它更贴近推理模型的真实工作方式，是 RAG 从"静态前置检索"走向"动态推理辅助"的关键一步。

### 6. OCR-Memory
**发生了什么**：这篇提出一种新型 agent memory：不是把历史压缩成文本摘要，而是把交互历史渲染为带索引锚点的图像，再通过"定位 + 转写"取回原文。这样可以减少摘要压缩导致的信息丢失，在 **Mind2Web** 和 **AppWorld** 上做到 SOTA。

**为什么重要**：它提供了不同于传统 summary memory 的路线，尤其适合长轨迹、强细节依赖的 agent 任务。
