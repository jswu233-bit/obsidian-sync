### 1. xAI 发布 Grok Build 0.1：面向 agentic coding 的高速编码模型

**发生了什么：**  
xAI 将 **Grok Build 0.1** 通过 xAI API 开放 public beta。这是一个专门面向 agentic coding 的模型，强调 Web 开发、调试、MCP 支持、工具调用与通用 agent 工作流；官方给出的速度是 100+ tokens/s，价格为输入 $1/M tokens、输出 $2/M tokens。来源：[xAI API / Grok Build 0.1 相关发布检索](https://x.ai/api)。

**为什么重要：**  
这条消息不是“又一个代码模型”那么简单，而是说明模型厂商已经把 coding model 的产品形态从“IDE 补全”推到“Agent harness 的默认引擎”。它明确提到可在 Grok Build、Cursor、OpenClaw、OpenCode 等 harness 中使用，本质上是在争夺“谁给长期执行型 Agent 供脑”。对于 Jamie 关注的 OpenClaw 和 Claude Code 生态，这类模型会直接影响 agent 成本、响应速度和可用性。

**Zoe点评：**  
📊 **短期看多，优先级上调。** Grok Build 0.1 值得进入测试池，尤其适合跑“低风险、高频、需要速度”的 coding/工具调用任务。但我不建议立刻替代主力模型：先用它做脚本生成、简单修复、UI 小工具，不要上生产级复杂链路。动作建议：如果明天有 OpenClaw 自动化小任务，可以拿 Grok Build 做一次对照测试。

### 2. MiniMax M3 发布：国产模型切入“长上下文 + 编码 Agent + 多模态”组合赛道

**发生了什么：**  
MiniMax 发布 **M3**，定位为面向 coding agents、长程自动化和多模态工作流的 frontier model。报道提到它支持 **100 万 token 上下文**、原生图像/视频输入，并与 MiniMax Code agent 产品绑定；官方称未来 10 天内会发布技术报告并开源对应权重。来源：[Startup Fortune - MiniMax M3 gives Chinese AI labs a new frontier coding test](https://startupfortune.com/minimax-m3-gives-chinese-ai-labs-a-new-frontier-coding-test/)。

**为什么重要：**

**Zoe点评：**  
📊 **谨慎看多，但等验证。** MiniMax M3 的方向非常对，但目前最大变量是：权重是否真开、benchmark 是否可复现、长上下文在真实任务里是否稳定。建议 Jamie 先关注“MiniMax Code + M3”是否能做跨页面/跨文档的稳定 PM 工作流，而不是只看榜单分数。短期不要押注，等技术报告出来再评估。

### 4. WaytoAGI 本周 AI 论文：SkillOpt 把 Skill 文档变成可训练参数

**发生了什么：**  
飞书知识库固定源《5.24-5.31｜本周顶级 AI 论文》重点收录了 **SkillOpt**。Microsoft Research 将自然语言 Skill 文档视为冻结智能体的可训练状态，通过多轮运行、反思和带验证门禁的文档编辑来优化 Skill；模型权重不变，优化对象是 SKILL.md 这种自然语言外部参数。来源：[WaytoAGI 飞书知识库](https://waytoagi.feishu.cn/wiki/Taqyw9f7hiMMLSk36BycHfjdn5b)，原帖：[DAIR.AI X thread](https://x.com/dair_ai/status/2061104052818108476)。

**为什么重要：**  
这和 OpenClaw 的 Skills 机制高度相关。过去很多团队写 skill/prompt 靠经验，SkillOpt 给出了一条更工程化的路线：把 skill 文档当成可迭代、可验证、可回滚的“外部参数”。论文显示它在多模型、多基准上都能带来明显提升，这说明 Agent 的上限不只取决于底层模型，也取决于我们能不能把行为规范写成可优化资产。

**Zoe点评：**  
📊 **强烈看多，建议立刻吸收进 OpenClaw 工作流。** 这条对 Jamie 最有用。我们以后写 SOP / SKILL.md 不应该只靠感觉，而要建立“任务样本 → 运行 → 失败分析 → 修改 skill → 留出验证”的闭环。动作建议：明天可以挑一个稳定任务，比如日报或搜索 SOP，做一次小型 SkillOpt 式改造。
