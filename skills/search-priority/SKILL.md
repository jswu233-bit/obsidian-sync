# search-priority

## 目标
统一搜索流程，避免工具使用顺序混乱，提升结果稳定性与可追溯性。

## 固定搜索顺序（强制）
1. **Pinchtab** - 浏览器自动化采集（优先）
2. **x-reader** - 多平台内容抓取
3. **multi-search-engine** - 多引擎聚合
4. **x-tweet-fetcher** - X / Twitter 专用
5. **Brave Search** - 通用网页搜索（兜底）

## 执行规则
- 任何“搜索类”任务，都必须先从第 1 步开始。
- 上游工具失败或结果不足时，按顺序降级到下一步。
- 禁止直接跳过前置工具（除非工具不可用）。
- 输出结果时必须标注来源链接。

## X / Twitter 特殊规则
- 若 X 内容无法直接读取，必须继续沿顺序降级，不允许以“无法获取”直接结束。
- 至少给出：
  - 原帖链接（若有）
  - 可验证替代来源（检索页/二手可信来源）
  - 明确说明信息完整度（完整/部分还原）

## 结果质量要求
- 结论要可验证（有链接）
- 多源交叉（关键结论至少2源）
- 禁止占位词：`未获取` / `待补` / `xxxxx` / `example.com`
