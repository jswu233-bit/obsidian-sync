# 4 角色协作系统 - 使用指南

## 系统架构

```
用户
  │
  ▼
🎯 Zoe (指挥官) - 我
  │
  ├──────┬──────┬──────┐
  │      │      │      │
 🧠军师 🔧工程师 ✨创作官 ✅检查官
 @军师  @工程师 @创作官 @检查官
```

## 角色说明

| 角色 | Bot | 职责 | 召唤场景 |
|------|-----|------|----------|
| **Zoe (我)** | @Zoe | 任务分配、进度协调、结果整合 | 所有任务入口 |
| 军师 | @军师 | 策略分析、方案评估、风险预判 | 需要决策建议时 |
| 工程师 | @工程师 | 技术执行、代码实现、系统维护 | 需要写代码/配置时 |
| 创作官 | @创作官 | 内容创作、表达优化、对外输出 | 需要写文案/报告时 |
| 检查官 | @检查官 | 质量审核、合规检查 | 需要审核内容时 |

## 使用方式

### 方式 1: 通过我协调
```
用户: @Zoe 我要做一个自动化备份系统

Zoe: 收到任务。让我召集团队：
@军师 请分析这个需求
@工程师 准备技术方案

军师: [分析结果]
工程师: [技术方案]

Zoe: 整合输出，交付用户
```

### 方式 2: 直接召唤特定角色
```
用户: @工程师 帮我写个 Python 脚本

工程师: [直接提供代码]

用户: @检查官 审核一下这段代码

检查官: [审核结果]
```

## 工作流程

### 技术项目
```
用户 → @Zoe
  ↓
@军师: 技术选型、架构设计
  ↓
@工程师: 代码实现
  ↓
@检查官: 代码审核
  ↓
@创作官: 技术文档（如需要）
  ↓
@Zoe: 整合交付
```

### 内容创作
```
用户 → @Zoe
  ↓
@军师: 内容策略
  ↓
@创作官: 内容创作
  ↓
@检查官: 内容审核
  ↓
@Zoe: 整合交付
```

只需要创建 **4 个 Bot**：

我（Zoe）继续使用现有的配置，作为协调者。

## 注意事项

- 我是用户的主要接口，负责协调 4 个角色

- 技术任务可以直接 @工程师
- 内容任务可以直接 @创作官
- 所有交付物建议 @检查官 审核
